翻譯暫時終止
QuickBASIC 一項一項地翻譯每一個程式行,一旦發生錯誤時,
便停止執行,然後停在反白的項目上,而這個錯誤通常就在該
項目左邊的位置上:

  a. 請立刻到反白項目的左邊,用交談框中所建議的其中一項
     作取代。

  b. 或, 更正任何反白項目左邊拼錯的關鍵字。

  c. 假若您想要比較有問題的結構與正確的結構時,請將游標
     移到該敘述內,並按下 F1 鍵,則正確的結構(或語法)將
     顯示在援助視窗中。

        * 不能暫停程式的執行。

    "監看點"(Watchpoint) 命令 --
        * 不能顯示變數的值。
        * 只有在當運算式變為真時,才會暫停程式執行。

"T/追蹤"命令(全功能表用)
開啟 "追蹤"(Trace On)命令,當您想要 QuickBASIC:
  * 以緩慢的動作來執行程式,並反白每一個所執行的敘述。
  * 在程式停止之前,記錄已被執行的最近 20 行程式碼。

SHIFT+F8 鍵來單步往回追溯, 然後再使用 SHIFT+F10 鍵來

"H/歷史"命令(全功能表用)
開啟 "歷史"(History On)命令開關,當您想要 QuickBASIC :
  * 以全速執行您的程式。
  * 記錄程式停止前的最近 20 行的程式碼。

然後您可以對 "歷史" 命令所記錄的 20 行程式碼向後(往回)和
向前追蹤,先按 SHIFT+F8 鍵單步往回追溯,然後再按 SHIFT+F10
鍵單步向前追縱,來檢查分歧和其他的程式動作。

秘訣: 使用 "歷史" 命令配合中斷點,來檢查您的程式中任何
      一組 20 個連續的敘述。





"B/中斷點切換"命令

  1. 移動游標到您想要中斷位置的那一行上。

  * 也可以增加變數到監看視窗上,然後從中斷點可以繼續向前單步

  ** "N/新增程式" 或 "O/載入程式"。

  3. 移動游標(或按 TAB 鍵)到您所要援助的項目上,然後按下
     F1 鍵。

  1. 從 "H/援助" 功能表選取 "內容"(Contents) 命令。
  2. 移動游標(或按 TAB 鍵)到您所要援助的項目上。
  3. 按 F1 鍵。

下方以反白顯示。





重新執行的提示
這個訊息常常在偵錯時發生,當您的程式暫停(例如,在中斷點),
您所修改的可能涉及到全部的程式,而不僅是只影響到所修改的
地方而已。例如 : 您可能重新定義陣列的維數(dimensions),
改變程序式的參數,或者編輯一些其他的宣告敘述。可以 :
  * 選取<確認>,持續此更改並且重新執行程式。
  * 選取<取消>,使得 QuickBASIC 對您的程式本文不作改變。
    假如重新執行程式會妨礙或影響到您的偵錯策略時,可以選擇
    此項。





新增主模組的交談框
所有被載入的模組皆出現在清單框中,選擇其中的一個成為主模組。

當您重新執行一個程式時,在主模組中第一個可執行的敘述,就是
第一個被執行的敘述。





儲存已被載入的檔案嗎?
您想要將記憶體中的內容清除,但是主記憶體中已經載入一個以上
的檔案, 這些檔案也已被修改過但尚未儲存時 :
  * 選取<是>,儲存檔案,然後執行清除記憶體的命令。假如其中
    有一個檔案尚未命名,系統將提示您輸入檔案名稱。
  * 選取<否>,清除記憶體,但不儲存檔案。
  * 選取<取消>,停止此命令,回到未選擇此命令前的狀態。





沒有主模組的提示
您企圖去執行一個主模組已被移出的程式時,會發生此項錯誤。

因為每一個程式必須有主模組,QuickBASIC 才能知道程式該從
那裡開始執行。您可以從下列兩者之一去著手 :
  * 從 "F/檔案" 功能表中,使用 "L/載入檔案" 命令,重新載入
    被移出的主模組。
  * 或從 "R/執行" 功能表中使用 "M/設定主模組" 命令,來指定
    目前載入模組中的一個模組當作主模組。

識別名稱的長度必須小於 40 個英文字元。





錯誤的識別名稱
識別名稱不能以 %, &, !, #, 或 $ 為結束(後置)字元。

以上的後置字元不允許出現在型態識別名稱,副程式名稱
或具名的(named) COMMON 區段。





識別名稱不能包含點號
使用者自定的型態識別名稱(type identifier),和記錄項目名稱
(record-element names)不能包括點號(".")。點號僅能當做您
記錄變數之分隔符號。

除此之外,一個變數名稱亦不能包含點號,假如點號是變數名稱
的一部份,可以在程式中使用 AS 子句來字義使用者定義的資料
型態。

假如在程式中的變數名稱有用到點號,且不是用 AS 子句來定義的
資料型態,您應該改成混合型態。例如,變數 ALPHA.BETA 應改為
AlphaBeta。





運算式太複雜
已超過 QuickBASIC 內部的限制。

例如,在計算運算式時,只給字串一個暫時的位置,與變數無關,
但是大量的此種字串會產生這種錯誤。另外,數值運算式中具有
許多複雜的次運算式亦會產生此項錯誤。

解決方法 :
  * 請簡化運算式。
  * 或, 將字串放到變數中。





不正確的數值
數值的格式無法對應到 QuickBASIC 所期望的數值格式中。

您可能產生打字上的錯誤。例如,數值 2p3 產生這項錯誤。

解決方法 :
    使用 QB 線上援助系統,選擇 "H/援助" 功能表的 "內容"
    命令,然後選擇關係鏈到 "資料型態" 的主題,以查閱正確的
    數值格式。





不正確的常數
不正確的運算式,例如 : 指定一個值給一個常數。

注意: 指定給常數的運算式可以包括除了指數(^)以外的數值
      常數,符號常數和任何一個算術式或者邏輯運算子。
      一個字串運算式指定給一個常數時,它只能由單一定字
      字串(literal string)所組成。





使用在 SUB, FUNCTION 或 DEF FN 範圍以外的錯誤
此敘述不允許在模組層次碼中使用。





使用在 SUB, FUNCTION 範圍以外的錯誤
此敘述不允許在模組層次碼或 DEF FN 函數中使用。





不正確的程序式或 DEF FN 函數
此敘述不允許在 SUB, FUNCTION, 或 DEF FN 函數內使用。





敘述不可用在 SUB 或 FUNCTION 定義之前
可以用在程序式定義之前的敘述只有 REM 和 DEF型態 敘述。





SUB 或 FUNCTION 敘述沒有對應的 END SUB 或 END FUNCTION 敘述
在程序式中缺少結尾的敘述。





END SUB 或 END FUNCTION 敘述沒有對應的 SUB 或 FUNCTION 敘述
在程序式中缺少 SUB 或 FUNCTION 敘述。
可能在編輯時被刪除掉。





END DEF 敘述沒有對應的 DEF 敘述
一個 END DEF 敘述沒有對應的 DEF 敘述。

同時檢查在 DEF...END DEF 結構中的其他控制結構,並且檢查他們
是否有正確的配對。例如, DEF...END DEF 結構中的 IF 敘述若沒有
對應的 END IF 敘述,則產生此項錯誤。






元素尚未定義
參考到尚未定義的使用者自定型態之元素。

例如,若使用者自定型態 MYTYPE 包含元素 A, B, 和 C, 然後試圖
使用變數 D 當作 MYTYPE 的元素,則會產生這個錯誤訊息。





型態未定義
在這模組中的使用者自定型態並沒有使用 TYPE...END TYPE 敘述
來定義。原因為:
  * 您可能拼錯使用者自定型態的名稱。
  * 或者可使用 "S/搜尋" 功能表的 "F/找尋" 命令來找尋在這
    模組中所有 TYPE...END TYPE 的定義。





必須是本行的第一個敘述
在 IF...THEN...ELSE 的區段結構中,僅在 IF, ELSE, ELSEIF 和
END IF 等敘述的前面可以有行號(line number)或標記(label),

在 SELECT...END SELECT 結構中,僅在 CASE 和 END SELECT 敘述
之前允許有行號或標記。





END SUB 或 END FUNCTION 必須是視窗中的最後一行
您試圖在程序式之後加入程式碼時,會發生此項錯誤。

解決方法 : 必須回到主模組(主程式),或開啟另一模組。






TYPE 敘述沒有對應的 END TYPE 敘述
TYPE 敘述沒有配對的 END TYPE 敘述。





END TYPE 敘述沒有對應的 TYPE 敘述
在 TYPE 宣告範圍以外使用 END TYPE 敘述。





在 TYPE 區段中有不正確的敘述
在 TYPE 和 END TYPE 敘述中僅允許有:
  * REM 敘述。
  * AS 資料型態子句。





不可在內含檔案中使用的敘述

解決方法 :
    從 "F/檔案" 功能表中使用 "M/合併" 命令,將內含檔插入

ALIAS, CDECL, 和 BYVAL 只能用於非 BASIC 的程序式中。

"FN" 只能使用在 DEF FN 函數名稱的頭二個英文字元。





此運作需要磁碟

$Metacommand 錯誤
語法不正確的假命令。

假命令必須加在註解中。

注意: $INCLUDE 假命令的語法如下:

  REM $INCLUDE: '檔案名稱'
  或
  ' $INCLUDE: '檔案名稱'

某些時候,不正確的巢狀(nested) IF 敘述會引起此項錯誤。

有正確的對應。例如 : IF...END IF 區段內的巢狀 IF 敘述

EXIT FOR 敘述使用在 FOR...NEXT 迴圈敘述之外。





EXIT 敘述不在 DO...LOOP 迴圈之內

資料空間不足
解決方法 :
請嘗試以下列方式修改資料所要求的空間 --
  1  在 OPEN 敘述中的 LEN 子句,請使用較小的緩衝區。
  2  使用 $DYNAMIC 假命令來產生一個動態陣列,因為動態陣列
     可以提供比靜態陣列還大的的資料量。
  3  使用一個固定長度的字串陣列來取代變動長度的字串陣列。
  4  儘量使用一個最小的資料型態來完成工作,且儘可能使用整數,
     以節省資料空間。
  5  儘量少用許多的小程序式,因為 QuickBASIC 在產生一個程序式
     時需要產生額外的空間來存放控制資訊。
  6  使用 CLEAR 敘述來修改堆疊(stack)的大小。只需使用足以
     完成您的工作所需的堆疊記憶體空間就夠了。
  7  程式中每一個敘述的長度不要超過 256 個位元組,若超過
     則 QuickBASIC 必須配置額外的本文緩衝區。
