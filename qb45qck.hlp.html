線上援助
  < 線上援助 >        < 內容 >        < 索引 >       < 產品支援 >
───────────────────────────────────────
  線上援助的使用方法如下:

    * 按下 F1 鍵 或觸動(click)滑鼠右鍵,可以隨時得到援助。
    * 在任何一個 BASIC 敘述,QuickBASIC 功能表項目或交談框上,
      按下 F1 鍵 時,就可以獲得援助。
    * 在援助視窗上可以隨時按下 ESC 鍵,將援助訊息由螢幕上移走;
      或者可以按下 PGDN 或 PGUP 鍵,來捲動螢幕上的援助訊息。

  所提供的援助資訊將顯示在 :
    * 螢幕頂端的援助視窗,或
    * 螢幕中間突出式的交談框。

  在援助視窗中也可以同時使用關係鏈來提供與此主題有關的其他援助,關係鏈包括:
    * 所有 BASIC 的關鍵字都是關係鏈。
    * 其他的關係鏈也可以被識別,因為 :
      - 都是用反白的角括弧包圍起來。
      - 而且,您可以按下 TAB 鍵來移動游標到一個關係鏈上。

  要顯示關係鏈上所參考的訊息,請將游標移到關係鏈上的任何位置,然後按下 F1 鍵
  或觸動滑鼠右鍵便可得到。

  秘訣:按 F6 或 SHIFT+F6 鍵 移動游標進入援助視窗,或移動滑鼠游標進入
       援助視窗並觸動左按鍵。

  QuickBASIC 會記錄最近 20 個所使用的關係鏈。您可以重覆 按 ALT+F1 鍵
  來回溯查閱。

  在援助視窗中,您也可以使用 QuickBASIC 功能表上的命令來 :
    * 複製本文(例如:使用 "C/複製" 命令來複製程式範例),然後離開援助視窗,
      再把它放到景象視窗中。
    * 在援助檔案中尋找您所需要的援助項目的其它用法。如果這個項目不是一個
      關係鏈,那將是非常有用的。

  F1 鍵和 ALT+F1 鍵兩者都是 "H/援助" 功能表命令的簡化鍵。按下 ALT+H 鍵
  拉下 "H/援助" 功能表,然後按下一個在 "H/援助" 功能表命令上符合您所要的
  反白英文字母鍵,來選取一個 "H/援助" 功能表中的命令。

  下面是 "H/援助" 功能表命令的摘要:

      命令           動作
    * 索引           在援助視窗中列出所有 BASIC 關鍵字的關係鏈。

    * 內容           在援助視窗中列出有關 BASIC 和 QuickBASIC 一套常用
                     的主題之關係鏈,例如: 顯示有關 BASIC 資料型態,或
                     運算式(expression)與運算子(operator)。

    * 主題:          顯示游標上所在項目的援助。

    * 線上援助       按下 SHIFT+F1 鍵 隨時顯示如何使用線上援助功能的
                     援助訊息。

AND 運算子 Quick 畫面
AND 運算子

語法
  結果 = 數值運算式 1  AND  數值運算式 2

AND 邏輯運算子(AND logical operator)對兩個數值運算式和對應的位元(bit)做
比較,若兩者對應的位元均為 1 時,其結果才會為 1。 AND 運算子的真值表
(Truth Table)如下 :

─────────────────────────────────
  第一個運算式位元          第二個運算式位元         結果之位元
─────────────────────────────────
           1                       1                      1
           1                       0                      0
           0                       1                      0
           0                       0                      0
─────────────────────────────────

使用函數或敘述時應注意的事項
  < 中文版使用時應注意的事項 >       < 內容 >      < 索引 >
───────────────────────────────────────
   QuickBASIC 對於印表機資料的輸出,有提供下面的敘述及函數 :

   (1). 可直接將資料傳送到印表機列印的敘述:

        LPRINT [運算式串列][{;|,}]
        LPRINT USING 格式字串; 運算式串列[{;|,}]

   (2). 調整印表機的列印寬度敘述:

        WIDTH LPRINT 寬度

   (3). 取得印表頭當時的行位置函數:

        LPOS(n)

    在 BASIC 程式中可以使用中文系統所提供的 PRINTER API(APPLICATION
    PROGRAMMING INTERFACE)命令碼,來控制列表機的列印效果。但 QuickBASIC
    中文版內的列表機資料輸出函數及敘述並不會解析這些 API 命令碼,即:
    這些 API 命令碼會被視為一般字元碼處理(例如: LPRINT 敘述)與計算
    (例如: WIDTH LPRINT 敘述,與 LPOS 函數), 假如 :

    *** 使用者先使用 WIDTH LPRINT 敘述設定一列的列印寬度;
    *** 而且又在 LPRINT 敘述內輸出 API 命令碼,例如 : 水平放大列印,
        則將來實際列印的列寬度與當初在 WIDTH LPRINT 敘述所設定的
        不符;當然 LPOS 函數所得的印表頭的行位置值也會不正確。

    所以,我們給使用者的建議是:

    *** 如果您所輸出的是純粹字元資料(不含 API 命令碼)時,可以很方便
        的使用這些敘述與函數,來得到正確列印結果。

    *** 如果您所輸出的資料中含有需要的 API 命令碼,請改寫成下面的方式,
        以得到正確的列印結果。

    例如: 在倚天中文系統下,經由 LPT1: 所接到的列表機,以水平兩倍放大
          列印 "中文化" 字元資料。

          OPEN "LPT1" FOR OUTPUT AS #1
          A$ = CHR$(&H7E) + "W2" + "中文化" + CHR$(&H7E) + "W1"
          PRINT #1,A$

(一). 中文碼的規格和用法 :

      QuickBASIC 中文版採用 BIG-5 碼,並且包含了中文系統所提供的造字區。
      對於 BIG-5 碼的高,低位元組的範圍(以十六進制值表示)如下:

      高位元組(High Byte): &H81 - &HFE
      低位元組(Low  Byte): &H40 - &H7E,  &HA1 - &HFE

      每一個中文碼可以用十六進制值表示的整數來定義。下例中,整數變數 A%
      內存放 "文" 字的 BIG-5 碼 &HA4E5,其中 &HA4 是高位元組(左邊部份),
      &HE5 為低位元組(右邊部份)。

           A% = &HA4E5

      字串變數可以包含中文字元碼,如下面的數個例子均表示字串變數 A% 內存放
      中文字串 "中文電腦" :

           A$ = "中文電腦"
           A$ = CHR$(164,164,164,229,185,113,184,163)
           A$ = CHR$(&HA4,&HA4,&HA4,&HE5,&HB9,&H71,&HB8,&HA3)

      同時在 QuickBASIC 工作環境中編輯程式,游標的移動是以 "字元" 為單位
      (將佔 1 個位元組長度的英文 ASCII 碼與佔 2 個位元組長度的中文碼,都
      當成一個字元)。


(二). 如何在中文版內使用表格圖形 :

     原英文表格圖形碼(Semi-Graphic Code,&H80 - &HFF),在目前中文版本中已被
     當成中文碼的高位元組(High-Byte),所以請改用中文符號碼顯示表格圖形;又
     由於中文字型顯示位置是英文字型的 2 倍,在更改程式時請同時注意相關位置
     的調整。
